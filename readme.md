# SentinelAI - Real-Time Fraud Detection and Prevention

A production-quality fraud detection system built end-to-end in a hackathon.
Trained XGBoost model, FastAPI inference backend, local LLM explanations via Ollama,
and a live dashboard that updates whenever you retrain the model.

---

## The Problem

Card fraud costs financial institutions over $32 billion annually.
Existing rule-based systems are rigid, easy to game, and produce too many false positives
that block legitimate customers. We wanted to build something smarter.

---

## What We Built

SentinelAI is a three-layer system:

**Layer 1 - Machine Learning**
We trained an XGBoost classifier on 284,807 real transactions from the
Kaggle Credit Card Fraud dataset. The dataset is extremely imbalanced
(only 0.17% fraud), so we used SMOTE to oversample the minority class during training.
The model outputs a fraud probability (0.0 to 1.0) on every transaction.

**Layer 2 - Decision Engine**
The probability is converted to a 0-100 risk score and routed through
a three-tier decision engine:
- Risk below 40: transaction approved
- Risk 40 to 75: step-up OTP authentication triggered
- Risk above 75: transaction blocked and flagged for review

**Layer 3 - Local LLM Explanation**
Every flagged transaction gets a plain-English explanation generated by a
local LLM (llama3 or mistral) running via Ollama. The model explains what
looks suspicious, why the risk level is what it is, what action is recommended,
and what a fraudster might be attempting. No external APIs. Everything runs on-device.

---

## Model Performance

After training on the full dataset:

| Metric    | Score  |
|-----------|--------|
| Accuracy  | ~99.9% |
| Precision | ~93%   |
| Recall    | ~88%   |
| F1-Score  | ~90%   |
| ROC-AUC   | ~98%   |

These metrics are written to model_meta.json at training time and served live
by the backend. The dashboard reads them on load and after every prediction,
so the numbers you see always reflect the current model.

## How to Run It

### Step 1 - Install dependencies

```bash
pip install -r requirements.txt
```

### Step 2 - Get the dataset (optional)

Download creditcard.csv from:
https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud

Place it in the backend/ folder. If you skip this, the training script
generates a synthetic dataset automatically so the full pipeline still runs.

### Step 3 - Train the model

```bash
cd backend
python sentinel_train.py
```

This trains the XGBoost model, prints evaluation metrics, and writes
model.pkl, scaler.pkl, and model_meta.json.

### Step 4 - Start Ollama (for LLM explanations)

```bash
ollama serve
ollama pull llama3
```

Get Ollama at https://ollama.ai.
The backend works without Ollama and falls back to a text explanation
if the connection fails.

### Step 5 - Start the backend

```bash
cd backend
uvicorn sentinel_main:app --reload --port 8000
```

API documentation: http://localhost:8000/docs

### Step 6 - Open the dashboard

Open frontend/sentinel.html directly in your browser,
or serve it locally:

```bash
cd frontend
python -m http.server 3000
```

Then visit http://localhost:3000/sentinel.html

## API

| Method | Path          | Description                                  |
|--------|---------------|----------------------------------------------|
| GET    | /health       | Health check                                 |
| GET    | /model/meta   | Returns live accuracy, F1, ROC-AUC from last train |
| POST   | /predict      | Runs inference, returns decision + LLM explanation |

The /model/meta endpoint re-reads model_meta.json from disk on every call.
This means if you retrain the model while the server is running, the dashboard
will pick up the new accuracy numbers automatically within 30 seconds.

### Sample POST /predict request

```json
{
  "amount": 2450.00,
  "time": 86400,
  "device": "Mobile App",
  "location": "Lagos, Nigeria",
  "V1": -2.3,
  "V14": -4.1,
  "simulate_fraud": false
}
```

### Sample response

```json
{
  "fraud_probability": 0.873421,
  "risk_score": 87.34,
  "decision": "BLOCK",
  "label": "High Risk",
  "message": "High fraud probability. Transaction blocked for review.",
  "explanation": "1. Suspicion Indicators: ...",
  "latency_ms": 142.3
}
```

- Vanilla HTML, CSS, JavaScript - no build step

---

## What Makes This Different

Most fraud detection demos hardcode risk scores or fake the model output.
SentinelAI runs a real trained model on every request. The risk score comes
from model.predict_proba(), not from a lookup table. The LLM explanation
is generated live by a local model, not a template. Retrain the model and
the dashboard shows the new accuracy in real time.
